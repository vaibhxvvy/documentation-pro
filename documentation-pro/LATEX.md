# LATEX.md — LaTeX Authoring System

## Purpose

This file defines the complete LaTeX setup for producing publication-quality PDFs
from documentation projects. It covers project structure, package selection, font
configuration, custom macros, compilation workflow, and error recovery.

Every PDF-targeted output from this skill is built using this LaTeX system.

## When to Use This File

Load this file when the output format is PDF. For Markdown-only or HTML-only outputs,
this file is not required. For hybrid outputs (Markdown source + PDF export), this
file defines the PDF compilation layer.

## Project Directory Layout

Every LaTeX book project uses this directory structure:

```
project-name/
├── book.tex               ← Root file: metadata, packages, includes
├── preamble.sty           ← Custom style file: all package configs and macros
├── frontmatter/
│   ├── titlepage.tex
│   ├── copyright.tex
│   ├── preface.tex
│   └── toc.tex            ← (generated by LaTeX, not handwritten)
├── chapters/
│   ├── ch01-introduction.tex
│   ├── ch02-[topic].tex
│   └── ...
├── appendices/
│   ├── appA-[name].tex
│   └── appB-[name].tex
├── backmatter/
│   ├── glossary.tex
│   └── index.tex
├── figures/
│   ├── ch01/              ← Per-chapter figure directories
│   └── ch02/
├── listings/
│   └── (code listing source files if using external files)
├── bibliography/
│   └── references.bib
├── scripts/
│   ├── build.sh
│   └── clean.sh
└── output/
    └── (compiled PDFs)
```

## Root File: book.tex

```latex
%% book.tex — Root document file
%% Compile with: latexmk -xelatex book.tex
%%               or: latexmk -lualatex book.tex

\documentclass[
  11pt,          % Body font size
  twoside,       % Two-sided layout for print
  openright,     % Chapters start on right-hand pages
  a4paper        % Paper size (use letterpaper for US)
]{book}

% Load custom preamble
\usepackage{preamble}

% Document metadata
\title{[Book Title]}
\author{[Author Name]}
\date{\today}
\hypersetup{
  pdftitle={[Book Title]},
  pdfauthor={[Author Name]},
  pdfsubject={[Subject]},
  pdfkeywords={[keyword1, keyword2, keyword3]},
}

\begin{document}

%% FRONT MATTER
\frontmatter
\include{frontmatter/titlepage}
\include{frontmatter/copyright}
\tableofcontents
\listoffigures
\listoftables
\lstlistoflistings
\include{frontmatter/preface}

%% MAIN TEXT
\mainmatter

% Optional: Part structure
% \part{Part I: Foundations}

\include{chapters/ch01-introduction}
\include{chapters/ch02-[topic]}
% ... additional chapters

%% BACK MATTER
\backmatter

\printglossaries
\printbibliography[heading=bibintoc]
\printindex

\end{document}
```

## Compilation: XeLaTeX vs LuaLaTeX

Use **XeLaTeX** (default): Faster compilation. Full OpenType font support. Best
compatibility with fontspec.

Use **LuaLaTeX** if: The project requires Lua scripting for dynamic content, or
advanced font features beyond what XeLaTeX provides.

Never use pdfLaTeX for new projects. pdfLaTeX does not support Unicode fonts
and requires separate encoding packages that are obsolete.

## Compilation Workflow

Use `latexmk` to manage compilation. `latexmk` automatically determines the correct
number of passes and runs biber/makeglossaries as needed.

```bash
# Standard compilation (XeLaTeX)
latexmk -xelatex -synctex=1 -interaction=nonstopmode book.tex

# LuaLaTeX compilation
latexmk -lualatex -synctex=1 -interaction=nonstopmode book.tex

# Watch mode (recompile on save)
latexmk -xelatex -pvc book.tex

# Clean build artifacts (keep PDF)
latexmk -c

# Clean everything including PDF
latexmk -C
```

The `.latexmkrc` configuration file:
```perl
# .latexmkrc
$pdf_mode = 5;                  # Use xelatex
$xelatex = 'xelatex -synctex=1 -interaction=nonstopmode -file-line-error %O %S';

# Run makeglossaries when needed
add_cus_dep('glo', 'gls', 0, 'makeglossaries');
add_cus_dep('acn', 'acr', 0, 'makeglossaries');
sub makeglossaries {
  system("makeglossaries $_[0]");
}

# Output to output/ directory
$out_dir = 'output';
```

## Multiple Compilation Passes

LaTeX requires multiple passes to resolve cross-references, bibliographies, and indices.
`latexmk` handles this automatically. Manual pass order for debugging:

```
Pass 1: xelatex book.tex          → generates .aux, .toc, .lof, .lot, .lol
Pass 2: biber book                → generates bibliography .bbl
Pass 3: makeglossaries book       → generates .gls, .acr
Pass 4: makeindex book            → generates .ind
Pass 5: xelatex book.tex          → resolves bibliography and glossary refs
Pass 6: xelatex book.tex          → resolves any remaining cross-references
```

## Error Recovery

Common LaTeX errors and their resolution:

| Error | Cause | Fix |
|---|---|---|
| `Undefined control sequence` | Missing package or typo in macro name | Check package is loaded in preamble.sty |
| `Missing $ inserted` | Math mode used outside math environment | Wrap in `$...$` or `\(...\)` |
| `Overfull \hbox` | Content exceeds line width | Add `\-` manual hyphenation, use `\allowbreak`, or rewrite sentence |
| `Font not found` | fontspec cannot locate the named font | Check font is installed system-wide |
| `Bibliography not found` | biber not run or .bib file path wrong | Check `\addbibresource{}` path; run biber manually |
| `Glossary entry not found` | \gls{} used before \newglossaryentry{} | Add entry to glossary.tex; run makeglossaries |
| `Too many floats` | Too many pending figures/tables | Add `\clearpage` to force flush floats |
| `Label multiply defined` | Same `\label{}` used twice | Make all label names unique |

## PDF Optimization

After compilation, optimize the PDF for distribution:

```bash
# Compress with Ghostscript (preserve quality for screen reading)
gs -sDEVICE=pdfwrite \
   -dCompatibilityLevel=1.7 \
   -dPDFSETTINGS=/ebook \
   -dNOPAUSE -dQUIET -dBATCH \
   -sOutputFile=output/book-optimized.pdf \
   output/book.pdf

# /screen   = 72 dpi (lowest quality, smallest file)
# /ebook    = 150 dpi (good for screen, small file)
# /printer  = 300 dpi (print quality)
# /prepress = 300 dpi + color profiles (commercial printing)
```

## Inter-file Relationships

```
LATEX.md
    ├── DIAGRAMS.md    (TikZ diagrams go in figures/ and are \input in chapters)
    ├── CODE.md        (lstlistings configuration extends the lstset in preamble.sty)
    ├── FIGURES.md     (figure placement rules align with LaTeX [H], [ht], [p] options)
    ├── TABLES.md      (table LaTeX code uses booktabs and tabularx from preamble.sty)
    ├── CITATIONS.md   (bibliography/references.bib is loaded by preamble.sty)
    ├── BUILD.md       (compilation commands are detailed in BUILD.md)
    └── EXPORT.md      (PDF output is the primary export; other formats use pandoc)
```
